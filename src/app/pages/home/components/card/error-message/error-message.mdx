## Error Message Component

The `ErrorMessageComponent` is used to display an error message along with a reload button for refreshing the content or handling errors.

### Usage

```
<app-error-message (restart)="sendReloadToParent()"></app-error-message>
```

### Properties

- `restart` (Output): An EventEmitter that emits a value when the reload button is clicked.

### Custom Styles

The component uses custom SCSS styles to style the error message and the reload button. You can customize these styles as needed in your application.


### Change Detection Strategy

This component utilizes the `ChangeDetectionStrategy.OnPush` strategy to optimize performance by ensuring changes are detected only when component inputs change.

### Example

```
<app-error-message (restart)="sendReloadToParent()"></app-error-message>
```

In this example, an error message is displayed, and when the "RELOAD" button is clicked, the `sendReloadToParent` method is called.

### Notes

- Be sure to provide content within the `<app-error-message>` tags to set the error message text.
- When the "RELOAD" button is clicked, it emits an event through the `restart` output, allowing you to handle the reload action in your parent component.

This component is designed to help you handle and display error messages and provides an option to trigger a reload action when necessary.
